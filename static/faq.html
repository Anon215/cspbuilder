Referrer policy

CSP standard version 1.1 includes a new directive referrer to instruct browser on Referer behaviour. Details of this directive are described in yet another standard Referrer Policy. The directive allows the following scenarios that are reflected in CspBuilder policy generation engine:

  • none — suppress Referer header completely
  • none-when-downgrade — suppress the header when moving from a TLS page to a plaintext page
  • origin — the header will only contain URL's base domain (origin), not the whole URL
  • origin-when-cross-origin — same as above, but only if moving to another website; browser will send full URL if moving with the same origin
  • unsafe-url — retain the default behaviour of sending full URL in Referer

By default, CspBuilder offers origin-when-cross-origin as default setting because it seems to provide the best balance between privacy and usability.
